## ЁЯзн How I work

> Most PMs write a doc and hand it off. I write the doc **and** build the v1.
> It keeps my specs honest, my estimates real, and my feedback loop short.

1. **Frame the problem** тАФ who's hurting, and why it matters.
2. **Spec it** тАФ a tight PRD with the smallest version worth shipping.
3. **Build it** тАФ a working prototype, usually an AI agent over a real API.
4. **Ship & learn** тАФ put it in front of users, read the feedback, iterate.
